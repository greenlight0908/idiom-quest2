## 5. 코드 수정 시 주의점

- `index.html`에는 과거 버전 함수와 뒤쪽의 최신 함수가 함께 남아 있다. 같은 이름의 함수가 두 번 있으면 **파일 뒤쪽 정의가 실제로 사용된다**. `drawAcademy`, `drawClassroom`, `drawHell`, `drawStatues`, `drawYeomra` 등을 수정할 때 검색 결과 중 뒤쪽 활성 정의를 수정해야 한다.
- 배경 화면 좌표와 이동용 32픽셀 논리 격자는 별개다. 캐릭터를 옮길 때 그림 좌표만 바꾸지 말고 충돌 좌표와 클릭 판정도 함께 바꿔야 한다.
- 새 실행 자산을 추가하면 `sw.js`의 캐시 배열에도 반드시 추가하고 `CACHE` 이름을 새 버전으로 올린다.
- `sw.js`의 `CORE`에 없는 파일은 오프라인에서 빠질 수 있고, 존재하지 않는 파일이 하나라도 들어가면 서비스 워커 설치가 실패할 수 있다.
- 글로벌 랭킹용 Google Apps Script 주소는 `index.html`의 `SERVER_URL`에 있다.

```bash
git push origin main
```

`main`에 푸시하면 GitHub Pages가 자동으로 다시 배포된다. 배포 후 화면이 이전 버전이면 강력 새로고침하거나 주소 뒤에 `?v=새커밋번호`를 붙인다.

## 8. 다음 작업 권장 순서

1. 새 한자성어 나라와 염라대왕, 석상 `뿅` 애니메이션을 실제 전체 플레이로 확인한다.
2. 모바일 화면에서 문·NPC·석상 클릭 판정을 확인한다.
3. `index.html` 안에 중복으로 남은 과거 그리기 함수를 정리해 잘못된 함수를 수정할 위험을 줄인다.
4. 사용하지 않는 v18·v19·v20 구버전 자산을 조사하되, `sw.js`와 코드 참조를 먼저 제거한 뒤 삭제한다.
5. 편의점 안젤라 접근 거리와 새 진열대 충돌을 실제 전체 플레이로 확인한다.
6. 역사 갈색 보도블록 길과 실제 이동 가능 경로가 직관적으로 일치하는지 확인한다.
7. 공개 전 실제 인물·학교·학원 로고 및 사진 기반 캐릭터 사용 범위를 최종 확인한다.

## 9. 현재 미확인 또는 주의할 부분

- 최신 한자성어 나라 v25와 염라대왕 v26은 정적 합성·문법 검사는 통과했지만, 사용자의 최종 전체 플레이 확인은 아직 필요하다.
- v28 편의점과 v29 역사도 정적 합성·문법·배포 검사는 통과했지만 모바일 포함 전체 동선 점검이 필요하다.
- `file://` 미리보기는 캐시 때문에 최신 파일이 보이지 않을 수 있다.
- 업로드용 폴더는 자동 동기화되지 않으므로 `최신본` 수정 후 반드시 다시 복사해야 한다.
- 홈페이지에서 내려받은 교사·시설 참고 사진은 `최신본/references/sidae-official/`에만 보관하며 GitHub에는 올리지 않았다.
